## 1. Product configurator — option-group handling
- **De-duplicate option groups by name.** The 4over `productsfeed` can repeat a
  group (e.g. business cards returned "Foil Color" / "Product Orientation"
  several times), which rendered duplicate rows and React duplicate-key errors.
  Added a `visibleExtraGroups` memo that de-dupes by group name.
- **Signs & Banners hide-list (not allow-list).** Instead of forcing a fixed
  field list, the configurator now takes `hiddenGroups`. For Signs & Banners we
  hide only Coating, Product Orientation, Flute Directions and H-Stakes —
  everything relevant to each product still shows. Rigid signs keep
  Size/Stock/Sides/Shape/Grommets/Quantity/Turnaround; banners now correctly
  show Grommets/Pole Pockets/Hems/D-Rings/Rope/Velcro. Hidden groups still keep
  their default option in the live price, so the quote stays correct.
- Business Cards and other categories keep their full option set.

## 2. Size grouping — one product, size in the calculator
The biggest change. Products that differ only by size were each shown as a
separate card (e.g. 10x10, 24x36 …). Now they are collapsed to **one card per
stock/type**, and the size is chosen in the price calculator.

- **Listing** (`print/[category]`): group products by size-stripped name → one
  card per stock. Examples: rigid signs 599 → 9, dual-raised BC 24 → 8,
  displays 11 → 3, roll labels → 5.
- **Configurator "size-variant" mode**: when a product has multiple size
  variants, the Size dropdown selects a product directly (its value IS a
  product_uuid) and the Size/Stock/Coating category cascade is skipped (stock is
  implied by the chosen card).
- **Product page** (`print/[category]/[typeSlug]`): builds the size variants
  (siblings with the same size-stripped name) and passes them; strips the size
  from the title/breadcrumb.
- Applied to all six print parents: Signs & Banners, Business Cards, Marketing
  Materials, Boxes & Packaging, Roll Labels & Stickers, Promo Products
  (via the shared `SIZE_GROUPED_PARENTS` list).

## 3. Size detection / grouping robustness
- **Strip size anywhere in the name** (not just a leading prefix). Some products
  carry the dimension mid-string ("Banner Stand With 11\" X 17\" …"). Switched
  `stripSize`/`extractSize` to match dimensions globally — this fixed the
  Displays grouping (11 → 3) and the Size dropdown there.
- **Trim trailing separators** so code-style names (boxes, e.g.
  "18PTC1S-CPBXNC-") don't end with a dangling dash.
- **Case-insensitive grouping key.** 4over descriptions vary in casing/spacing
  ("With" vs "with", "OVAL" vs "Oval"), which split groups into near-duplicate
  cards. Group by a normalized (lowercased, single-spaced) key on both the
  listing and the product-page size matching. Stickers: 11 → 9.

## 4. Product title
- For grouped categories, the leading/embedded size is removed from the product
  heading and breadcrumb (size is in the calculator now). E.g.
  "10\" X 10\" 10mm White Coroplast" → "10mm White Coroplast".

## 5. Deployment / build robustness
- **Category landing pages no longer require Supabase.** `/print/<parent>`
  (e.g. /print/signs-banners) is pure static cards but used to call
  `createClient()` up-front, throwing a server-side exception on deployments
  without DB env vars. Moved `createClient()` into the product-listing branch.
